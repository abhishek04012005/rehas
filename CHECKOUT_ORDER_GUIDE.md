# Checkout & Order Management Guide

## Overview

When users click "Proceed to Checkout" on the product detail page, they are taken to a comprehensive checkout form where they can enter their details. After successful submission, an order is created with a unique **Order ID** that is displayed prominently before payment.

## Checkout Flow

```
Product Page → Click "Proceed to Checkout" 
    ↓
Checkout Form (Enter Name, Email, Phone, Address)
    ↓
Order Created Successfully (Show Order ID)
    ↓
Payment Gateway (Razorpay/COD)
    ↓
Payment Success/Failed
```

## Order Number Generation

The **Order ID** is automatically generated by the database as a unique `BIGSERIAL` primary key. Each order gets a sequential number like `#123456`.

### Example Order ID Display
```

#5487392
```

## Database Schema

### Orders Table

```sql
CREATE TABLE IF NOT EXISTS orders (
  id BIGSERIAL PRIMARY KEY,
  
  -- Customer Information
  full_name VARCHAR(255) NOT NULL,
  email VARCHAR(255) NOT NULL,
  phone_number VARCHAR(10) NOT NULL,
  birth_date DATE,
  age INT,
  
  -- Product/Service Information
  product_title VARCHAR(255) NOT NULL,
  order_type VARCHAR(50) NOT NULL,
  service_id VARCHAR(255),
  service_title VARCHAR(255),
  service_description TEXT,
  
  -- Address Information
  address_line_1 VARCHAR(500),
  address_line_2 VARCHAR(500),
  city VARCHAR(100),
  state VARCHAR(100),
  postal_code VARCHAR(20),
  country VARCHAR(100) DEFAULT 'India',
  
  -- Payment Information
  amount DECIMAL(10, 2) NOT NULL DEFAULT 0,
  currency VARCHAR(3) DEFAULT 'INR',
  amount_in_paise BIGINT,
  
  -- Payment Fields
  payment_status VARCHAR(50) DEFAULT 'pending',
  payment_method VARCHAR(50),
  transaction_id VARCHAR(255),
  razorpay_order_id VARCHAR(255),
  razorpay_payment_id VARCHAR(255),
  razorpay_signature VARCHAR(255),
  
  -- Order Status
  status VARCHAR(50) DEFAULT 'pending',
  notes TEXT,
  admin_notes TEXT,
  
  -- Timestamps
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### Key Fields

| Field | Type | Purpose |
|-------|------|---------|
| `id` | BIGSERIAL | **Order ID** - Unique identifier |
| `full_name` | VARCHAR | Customer name |
| `email` | VARCHAR | Customer email |
| `phone_number` | VARCHAR | 10-digit phone (starts with 6-9) |
| `product_title` | VARCHAR | Name of product/service ordered |
| `amount` | DECIMAL | Order amount in rupees |
| `address_line_1` | VARCHAR | Street address |
| `city` | VARCHAR | City |
| `state` | VARCHAR | State |
| `postal_code` | VARCHAR | Postal/Zip code |
| `payment_status` | VARCHAR | unpaid, pending, paid, failed |
| `order_type` | VARCHAR | service, course, product |
| `created_at` | TIMESTAMP | Order creation time |
| `status` | VARCHAR | pending, confirmed, processing, completed, cancelled |

## SQL Queries

### Get Order Details by ID
```sql
SELECT * FROM orders WHERE id = YOUR_ORDER_ID;
```

### Get All Orders by Customer Email
```sql
SELECT * FROM orders 
WHERE email = 'customer@example.com' 
ORDER BY created_at DESC;
```

### Get All Pending Orders
```sql
SELECT * FROM orders 
WHERE payment_status = 'pending' 
ORDER BY created_at DESC;
```

### Get Orders by Date Range
```sql
SELECT * FROM orders 
WHERE created_at BETWEEN '2024-01-01' AND '2024-12-31' 
ORDER BY created_at DESC;
```

### Get Revenue Analysis
```sql
SELECT 
  DATE(created_at) as order_date,
  COUNT(*) as total_orders,
  SUM(amount) as total_revenue,
  AVG(amount) as avg_order_value
FROM orders
WHERE payment_status = 'paid'
GROUP BY DATE(created_at)
ORDER BY order_date DESC;
```

### Get Orders by Product
```sql
SELECT * FROM orders 
WHERE product_title = 'Rudraksha Bracelet' 
ORDER BY created_at DESC;
```

## API Endpoints

### Create Order
**POST** `/api/orders/create`

**Request Body:**
```json
{
  "fullName": "John Doe",
  "email": "john@example.com",
  "phoneNumber": "9876543210",
  "addressLine1": "123 Main Street",
  "addressLine2": "Apt 4B",
  "city": "Mumbai",
  "state": "Maharashtra",
  "postalCode": "400001",
  "country": "India",
  "productTitle": "Rudraksha Bracelet",
  "amount": 999,
  "orderType": "product"
}
```

**Response:**
```json
{
  "success": true,
  "orderId": 5487392,
  "message": "Order created successfully"
}
```

### Error Responses
- **400**: Missing required fields
- **400**: Invalid email/phone format
- **500**: Database error
- **503**: Database not initialized

## State Management

Orders are managed through:
- **CheckoutContext**: Stores product data (`productTitle`, `amount`)
- **Component State**: Form data, validation errors, order confirmation

## Validation Rules

1. **Full Name**: Required, non-empty
2. **Email**: Required, valid email format
3. **Phone Number**: 
   - Required
   - Exactly 10 digits
   - Must start with 6, 7, 8, or 9
4. **Address** (For Products):
   - Address Line 1: Required
   - City: Required
   - State: Required
   - Postal Code: Required

## Database Indexes

For performance optimization, indexes are created for:
- `status` - For filtering by order status
- `created_at` - For sorting by date
- `email` - For customer lookups
- `phone_number` - For customer verification
- `payment_status` - For payment reconciliation
- `order_type` - For service/product filtering

## Troubleshooting

### Order Not Created
- Check if all required fields are filled
- Verify phone number format (10 digits, starts with 6-9)
- Check database connection in Supabase

### Order ID Not Showing
- Ensure payment form renders after confirmation
- Check browser console for errors
- Verify API response contains `orderId`

### Form Validation Errors
- Phone number must be exactly 10 digits
- Email must be valid format
- Address fields required for products only

## Support

For issues with orders or checkout:
1. Check browser console for errors
2. Verify database table exists (see DATABASE_SETUP.md)
